# SS1：信息完整性智能校验

**Skill ID**：`SS1`  
**核心定位**：校验 SS0 输出的结构化数据，输出 JSON 格式补全建议供前端弹窗展示。  
**前置依赖**：[SS0] 结构化采访数据。

## 精确落地流程

1. 扫描**必填项**：姓名、生卒日期、至少 1 条核心记忆、1 位主致辞人、1 张肖像照。
2. 扫描**高优先级选填项**：遗愿、至少 3 张不同时期照片、语音样本。
3. 扫描**普通选填项**：更多照片/录音、特殊经历、多位致辞人等。
4. 按优先级排序，最多输出 3 条补全提示；每条说明补全后的效果。
5. 若必填齐全，返回 `ready`；若非必填缺失，提供「跳过补全，使用默认值」选项。

## 输出规范（仅输出合法 JSON）

```json
{
  "status": "missing_details",
  "can_skip": true,
  "prompts": [
    {
      "field": "core_memory",
      "priority": "high",
      "ui_prompt": "请补充一个最能代表爷爷性格的具体小事（例如：每天早上都会为家人煮一碗热粥），我们会将其作为核心分镜，让视频更有温度",
      "default_value": "他一生勤劳朴实，默默为家庭付出"
    },
    {
      "field": "voice_sample",
      "priority": "medium",
      "ui_prompt": "如果有爷爷的语音或视频录音，请上传一段，我们可以克隆他的声音，让数字人更真实",
      "default_value": "使用系统默认老年男性音色"
    }
  ]
}
```

**状态说明**：`status` 可为 `ready` 或 `missing_details` 等枚举，与前端约定一致即可。
