# 追悼会大屏数字人短片 · Skills 交付包（MPO v5.0）

## 甲方边界（子 Skill）

**子 Skill（`MV01`～`MV06`）只负责**：基于 **用户信息与生活内容**，产出 **视频脚本、叙事模板、分镜口播、画面文字描述与成片节奏脚本**。

**子 Skill 不负责**：后端、API、数据库、任务队列、文件存储、FFmpeg 等。工程实现可另册维护（如 `平台集成与数据编排说明.md`）。

## 包内文件说明

| 文件 | 说明 |
|------|------|
| `mother-skill.md` | **母 Skill**：产品向总调度与闸门（内容边界见内文 §1） |
| `MV01-采访与素材采集.md` ~ `MV06-成片剪辑与交付.md` | **内容子 Skill**（1→6） |
| `MV04-语音试听+人物场景道具三库锁定.md` | **重点**：语音试听 + 人物/场景/道具三库的锁定与绑定（G4） |
| `平台集成与数据编排说明.md` | **工程文档（可选）**：合成/校验/API 提示（不属于子 Skill） |

## 管线顺序（摘要）

1. **MV01** 采访与素材采集 → **G1**  
2. **MV02** 信息校验与总结反馈 → **G2**  
3. **MV03** 分镜、提示词与口播文稿（叙事模板核心）→ **G3**  
4. **MV04** 人物/场景/道具锁定（圣经）与试听 → **G4**  
5. **MV05** 朗读稿细化与画面叙事说明 → **G5**  
6. **MV06** 成片节奏与轨级脚本结构 → **G6**

**范围**：现场大屏数字人纪念短片（非整场仪式编排）。

## 使用建议（供接入方）

1. **内容模型**：加载母 Skill + 当前步 MV；勿把《平台集成》全文塞进内容 System Prompt。  
2. **工程团队**：以《平台集成》对接存储、状态机、合成与校验。  
3. **输出**：子 Skill 产出以 **合法 JSON** 为主（无 Markdown 围栏、无注释）。
