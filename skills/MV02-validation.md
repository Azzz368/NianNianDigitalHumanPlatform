# MV02：信息校验与总结反馈

**步骤序号**：`2 / 6`  
**Skill ID**：`MV02`  
**适用范围**：大屏数字人短片管线。

## 本子 Skill 的范围（与甲方一致）

只产出 **能否支撑写脚本** 的校验结论、补全建议与给人看的总结文案；**不写**审核工单状态机、数据库或接口（见 `平台集成与数据编排说明.md`）。

## 核心定位

校验 **MV01** 结构化数据是否满足成片最低要求；输出补全建议与**可读总结**，供家属做「总结反馈」——确认材料足以支撑分镜与口播。

## 前置依赖

**MV01** 输出，且 **G1 已通过**。

## 人类闸门 G2

家属对校验结果与总结表态（通过 / 补全后再跑 MV02 / 返回 MV01 补采）。通过后方可进入 **MV03**。

**局部循环**：可只更新总结文案或只重开**一条**补全提示对应字段，未点名项保持已定稿。

## 落地流程

1. **必填**：姓名、生卒年、至少 1 条核心记忆、1 张可用肖像、目标成片时长意向（或由系统给默认，写入 `target_duration_sec` 建议值）。
2. **高优选填**：语音样本（供 MV04 复刻）、多条不同时期照片。
3. 最多 3 条补全提示，说明补全后对成片的影响。
4. 生成一段给人读的「家属可见总结」；若仅输出 JSON，可将 `summary_for_human` 放入字符串字段，由产品在界面展示。

## 输出规范（仅输出合法 JSON）

```json
{
  "status": "ready",
  "can_skip_optional": true,
  "target_duration_sec_suggestion": 300,
  "summary_for_human": "已具备肖像与一条核心记忆，建议补充约60秒语音以便声音复刻更稳。",
  "prompts": []
}
```

**状态说明**：`status` 可为 `ready` 或 `needs_input`；`needs_input` 时 `prompts` 为非空数组，字段与产品/工程约定即可。
